1. What is the difference between cleaning and shaping data?

| Aspect                | Cleaning Data                                                       | Shaping Data                                                                  |
| --------------------- | ------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| Purpose               | Improves data accuracy and quality                                  | Transforms data for better analysis                                           |
| Focus                 | Fixes errors and inconsistencies                                    | Restructures and organizes data                                               |
| Common Tasks          | Removing duplicates, correcting data types, handling missing values | Filtering rows, splitting columns, adding calculated fields, appending tables |
| Outcome               | Reliable and error-free data                                        | Structured and analysis-ready data                                            |
| Example from Practice | Correcting data types and removing inconsistencies                  | Splitting Order Date and appending January and February datasets              |

Data cleaning ensures that the dataset is accurate, consistent, and free from errors. In contrast, 
data shaping restructures the data to make it more meaningful and suitable for analysis.
Cleaning prepares the data, while shaping enhances its usability for insights and reporting.

3. Check-in: Walk Through One Transformation You Did and Why

Transformation: Appending Two Tables

Objective: Combine the January and February sales datasets into a single unified table.

📂 Datasets Used

Online_Sales_January.csv
Online_Sales_February.csv

Steps Performed in Power Query

Imported both CSV files into Power BI.
Opened the Power Query Editor.

Navigated to Home → Append Queries → Append Queries as New.
Selected both tables.
Clicked OK.
Renamed the new table to Online_Sales_2024.
Clicked Close & Apply.
📊 Before and After

Before Appending

Dataset 	Number of Rows
January Sales   	10
February Sales	        10

After Appending

Dataset 	Number of Rows
Online_Sales_2024	20

 Why This Transformation Was Useful

It consolidated data from multiple sources into a single dataset.

It enabled comprehensive analysis across different months.

It simplified reporting and dashboard creation.

It reflected real-world business scenarios where data is stored in separate files.
