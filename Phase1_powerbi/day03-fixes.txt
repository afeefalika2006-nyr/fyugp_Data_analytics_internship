1. What kinds of “mess” did you find in the data?

The dataset contained several common data quality issues:

Missing Values
Example: Blank Order ID in the "Incomplete Record" row.
Duplicate Records
Example: Duplicate entries for Order ID 1024.
Incorrect Data Types
Example: The value "Two" in the Quantity column instead of the numeric value 2.
Example: "$120" in the Unit Price column.
Inconsistent Date Formats
Example: Dates appeared as both 01-01-2024 and 02/01/2024.
Inconsistent Text Formatting
Example: "electronics" vs. "Electronics" and "new york" vs. "New York".
Extra Spaces
Example: Customer names with leading or trailing spaces.
Missing Calculations
Example: Blank values in the Total Amount column.

These issues made the dataset unreliable for analysis before cleaning.

2. Why do you think real-world data is often messy?

Real-world data is often messy due to several reasons:

Manual Data Entry Errors
Typos, incorrect formats, and missing values occur when data is entered by humans.
Multiple Data Sources
Data collected from different systems often lacks standardization.
Lack of Validation Rules
Without proper constraints, inconsistent values can be recorded.
System Limitations and Integration Issues
Different software systems store and export data in varying formats.
Data Updates and Deletions
Cancelled transactions and removed records create gaps and inconsistencies.
Unstructured Business Processes
Organizations may not follow uniform data entry standards.

These factors highlight why data cleaning is a crucial step before analysis and visualization.
